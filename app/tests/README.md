# Тесты (MVP)

Эта структура фиксирует правила работы со слоями тестов.

- Builders-first: данные и сценарии собираем через builders.
- Слои тестирования: Unit, Integration, Functional.
- Builders/Shared — общие builders для всех слоёв.
- В Integration/Functional работаем с БД явно: persist/flush/clear.
- Моки в домене запрещены (domain должен быть реальным).
- Модули пока отсутствуют, структура едина для всего приложения.
- Новые тесты добавляем только в соответствующий слой.
- Общий инфраструктурный код — в tests/_support.
