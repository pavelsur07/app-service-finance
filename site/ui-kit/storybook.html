01 — Decisions

Решения по всем расхождениям из аудита. Каждое решение — одна строка с обоснованием. PROPOSED — требует подтверждения.

1
Card padding
16px 20px
Компромисс между плотными KPI и просторными drawer-секциями; не теряет содержимое мелких карточек и не выглядит «дёшево» в крупных.
2
KPI value size
22px (compact) · 28px (hero)
Две шкалы: 22 — для рядов виджетов, 28 — для одиночного hero-метрика (общий баланс, главный остаток). 26 удалён.
3
Border color
#E8ECF3
Используется чаще, мягче по тону; #E2E8F0 удалён. Inputs и cards теперь единая линия.
4
Drawer width
680px
Достаточно для двухколоночных форм (split-операции, parent + child fields); 640 ужимает.
5
Backdrop
opacity 0.45 · blur 6px
Среднее затемнение, мягкий blur — модалки читаемы, контекст за ними различим.
6
Avatar / initial sizes
sm 24 · md 32 · lg 40
Шаг кратный 8. Прежние 28/30/34 убраны — нет смысла, разница неразличима.
7
Direction indicator
colored dot + знак суммы
Минимум визуального шума, не конкурирует со status-пилюлями. Знак направления передаётся самой суммой (+/−).
8
Status
pill с точкой (везде)
Пилюля заметнее dot+text, лучше для accessibility (форма + цвет). Sync-статусы на счетах тоже теперь pill.
9
Money — знак минуса
U+2212 (−) · везде
Типографически правильный, ширина совпадает с +. Hyphen-minus запрещён в формате сумм.
10
Money — миллионы
М (короткая запись)
В таблицах и на осях графиков: компактнее «млн», вписывается в узкие колонки. В описаниях допустимо «млн ₽».
11
Money — цвет
цвет только для дельт
Балансы — всегда тёмные (нейтральные). Зелёный/красный — только для %-дельт, сумм поступлений/расходов и signed transactions. Остаток на конец периода — НЕ цветной.
12
Tint chip
именованные токены
Никакой математики с opacity — каждый тип чипа имеет свой набор переменных (--src-api-bg, --status-success-bg, …). Предсказуемо, не зависит от подложки.
13
Drawer body padding PROPOSED
24px 28px
Чуть просторнее основной страницы, передаёт сосредоточенность задачи. Подтвердите.
14
Tabular numerals PROPOSED
для всех числовых ячеек таблиц + сумм
font-variant-numeric: tabular-nums по умолчанию для денег и счётчиков. Не для процентов в KPI-карточках (там вес доминирует).
15
Icon color rule
currentColor inheritance · semantic роль на пункте
Иконки внутри компонентов наследуют currentColor. Цветом окрашивается ВЕСЬ пункт по семантической роли (default · primary · danger), не иконка отдельно. Эмодзи-иконки запрещены.

Правила (must)

  • Знак минуса — U+2212 (−). Не U+002D (-).
  • Разделитель разрядов — U+2009 (&thinsp;), не обычный пробел.
  • Десятичные — запятая ,. Только в коротких форматах и процентах.
  • «млн» и «тыс.» — нет. Используем М и К для коротких форм.
  • ₽ — суффикс, всегда после числа с пробелом. $/€ — префикс без пробела.
  • Цвет: signed transactions, deltas — да; балансы, итоги по дню, остаток на конец — нет.
  • Шрифтовое начертание — font-variant-numeric: tabular-nums в любой ячейке таблицы или блоке суммы.
  • Если значение ноль и контекст денежный — приглушённый --text-faint, цифра 0 или em-dash — ₽.
