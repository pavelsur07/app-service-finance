# Рефакторинг маппинга детализированных отчётов Wildberries

В рамках задачи расширена и упрощена работа с полями сумм в маппинге детальных отчётов:

- Добавлен единый провайдер источников сумм, который хранит список доступных сырых полей (retail_price, acquiring_fee, cashback_amount, deduction, логистическое возмещение и др.) и описания для интерфейса.
- Контроллер маппинга использует провайдер для построения выпадающего списка, чтобы добавление новых полей не требовало доработки формы.
- Генератор недельного PnL получает значения через тот же провайдер, поэтому новые поля автоматически обрабатываются при расчётах.

Такой подход исключает дублирование логики, ускоряет добавление новых вариантов (например, для разных типов документов и обоснований) и делает поведение маппинга и отчётов согласованным.
