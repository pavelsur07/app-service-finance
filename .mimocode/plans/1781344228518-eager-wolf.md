# Plan: Migration of Legacy Classes to Modules

## Context

Legacy directories (`src/Service/`, `src/Repository/`, `src/Controller/`, `src/Exception/`, `src/Message/`, `src/MessageHandler/`, `src/Command/`, `src/DataFixtures/`) contain ~41 PHP files that need migration to proper modular locations. CLAUDE.md marks these zones as forbidden for new code. Entities have already been migrated to their modules; this plan covers the remaining service/controller/message/command layers.

**Note:** `src/Entity/` is empty on disk — legacy entities (Document, PLCategory, etc.) have already been moved to `Finance/Entity/`, `Company/Entity/`, etc.

## Per-File Migration Checklist

For each file being moved:
- [ ] Create target directory if it doesn't exist
- [ ] Move file, update `namespace` declaration
- [ ] Update all `use App\OldNamespace\ClassName` in importing files
- [ ] If Message: update `config/packages/messenger.yaml` routing
- [ ] If Controller: update `config/routes.yaml` routing
- [ ] If Entity/Repository: update `config/packages/doctrine.yaml`
- [ ] Run `make codex-prepare && make codex-test-unit`
- [ ] Verify no remaining `use App\OldNamespace\` references
- [ ] Delete old file

## Config Changes Required

### `config/packages/messenger.yaml`

```yaml
# Before:
App\Message\ApplyAutoRulesForTransaction: async_pipeline
App\Message\EnqueueAutoRulesForRange: async_pipeline
App\Message\SendRegistrationEmailMessage: async_sync

# After:
App\Cash\Message\ApplyAutoRulesForTransaction: async_pipeline
App\Cash\Message\EnqueueAutoRulesForRange: async_pipeline
App\Company\Message\SendRegistrationEmailMessage: async_sync
```

### `config/routes.yaml`

```yaml
# Add if not present:
security_controllers:
    resource:
        path: ../src/Shared/Controller/
        namespace: App\Shared\Controller
    type: attribute

company_controllers:
    resource:
        path: ../src/Company/Controller/
        namespace: App\Company\Controller
    type: attribute
```

Finance, Notification, and Marketplace controllers should already have route config if those modules have existing controllers.
